virtuoso数据库使用总结

  • RDF
  • virtuoso
  • 数据库

最近在virtuoso使用过程中遇到三个错误，数据库启动失败、 上传数据文件失败和执行更新操作的sparql语句失败。

1.数据库启动失败，错误信息：

15:19:37 The transaction log file has been produced by server version ‘07.20.3214’. The version of this server is ‘07.10.3207’. If the transaction log is empty or you do not want to replay it then delete it and start the server again. Otherwise replay the log using the server of version ‘07.20.3214’ and make checkpoint and shutdown to ensure that the log is empty, then delete it and start using new version.
解决办法：可以阐释删除掉锁文件，锁删除rm -rf /数据存放路径/virtuoso/db/virtuoso.lck 。

2.上传数据时，上传数据文件失败，错误信息：Error 42000: [Virtuoso Driver][Virtuoso Server]FA003: Access to ‘/root/post-install’ is denied due to access control in ini file

解决办法：因为存放路径权限限制问题导致上传数据失败，需要把要上传的数据文件，放到virtuoso.ini配置文件中DirsAllowed配置项指定的目录下。

3.执行更新操作的sparql语句时，执行失败。

解决办法：virtuoso对更新操作的sparql语句有权限限制，不同连接方式下需要设置相应的权限，下面介绍常用的两种方式。
方式一，通过http访问时（如：http://10.0.10.14:8890/sparql）：
第一步，打开该页面，并点击选择spraql用户

第二步，打开下面页面，选择相应权限，添加到右面，保存提交即可

方式二，通过tcp访问时（如：10.0.10.14:1111，使用默认dba账户登录），选择dba账户，通过方式的方法即可实现权限赋予。
